### TC-001: 정상 회원가입

**테스트 날짜**: 2025-11-30  
**테스트 환경**: 웹 (Chrome)  
**상태**: 🟡 부분 통과

**전제조건**:
- ✅ 사이트 접속 확인
- ✅ 로그인 페이지 접근 확인
- ✅ 회원가입 페이지 접근 확인

**테스트 단계**:
1. ✅ 로그인 페이지에서 "회원가입" 버튼 클릭
2. ✅ 회원가입 폼 표시 확인
3. ✅ 이메일 입력 필드 확인
4. ✅ 비밀번호 입력 (8자 입력 시도, 입력 확인됨)
5. ✅ 비밀번호 확인 입력 (8자 입력 시도, 입력 확인됨)
6. ⚠️ 약관 동의 체크박스 클릭 시도 (체크 상태는 확인되었으나 실제 동작 확인 필요)
7. ⚠️ 회원가입 버튼 클릭 시도 (클릭은 되었으나 결과 확인 제한)

**예상 결과**:
- 회원가입 성공 메시지 표시
- 로그인된 상태로 메인 페이지 이동

**실제 결과**:
- ✅ 회원가입 폼 UI 정상 표시
- ✅ 입력 필드 정상 작동 (비밀번호 필드 확인됨)
- ✅ 약관 체크박스 UI 정상 표시
- ⚠️ 회원가입 완료 여부 확인 필요 (자동화 제한으로 실제 제출 결과 확인 불가)

**비고**: 
- 브라우저 자동화로 UI 요소와 입력 필드는 확인되었으나, 실제 회원가입 제출 프로세스 완료 여부는 수동 확인 필요
- 이메일 입력이 일부 세션에서 제대로 반영되지 않을 수 있음 (Flutter 웹 렌더링 특성)

**권장 조치**: 수동 테스트로 실제 회원가입 완료 및 이메일 인증 플로우 확인

**상태**: ⏸️ 전체 플로우 테스트

**전제조건**:
- ⚠️ TC-001 완료 (회원가입)
- ⚠️ TC-008 완료 (로그인)

**테스트 단계**:
1. ⏸️ 로그인
2. ⏸️ 주소 검색 (TC-025)
3. ⏸️ 주소 선택
4. ⏸️ 공인중개사 검색 (TC-039)
5. ⏸️ 다중 견적 요청 (TC-053)
6. ⏸️ 중개사 답변 (TC-067, TC-068)
7. ⏸️ 견적 비교 (선택사항)

**예상 결과**:
- 전체 플로우가 끊김 없이 진행
- 각 단계에서 적절한 피드백 제공
- 데이터가 올바르게 저장되고 전달

**실제 결과**:
- ⏸️ 종합 플로우 테스트 필요

**비고**: 이 테스트는 전체 시스템의 통합 테스트입니다.

**수동 테스트 가이드**:
1. **회원가입 및 로그인**
   - 새 계정 생성
   - 로그인 확인

2. **주소 검색 및 선택**
   - 메인 페이지에서 주소 검색
   - 주소 선택
   - 부동산 정보 자동 조회 확인

3. **공인중개사 검색**
   - "공인중개사 찾기" 버튼 클릭
   - 검색 결과 확인
   - 필터링 테스트

4. **다중 견적 요청**
   - 여러 중개사 선택
   - 견적 요청 폼 작성
   - 제출

5. **중개사 답변 (별도 계정/브라우저)**
   - 관리자 대시보드에서 링크 확인
   - 링크 클릭하여 답변 페이지 접근
   - 답변 작성 및 제출

   - 사용자 계정으로 로그인
   - "내집관리" → "견적 이력" 확인

   - 견적 비교 기능 테스트 (있는 경우)
